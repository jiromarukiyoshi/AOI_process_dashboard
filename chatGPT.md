# AOI長泉倉庫進捗ダッシュボード Webアプリ開発仕様書

## 1. 目的

AOI長泉倉庫の作業進捗をWebアプリで可視化する。

FFC出荷日単位で対象データを表示し、
FFC出荷NO単位で集合梱包・釣銭機系の作業進捗および全体完了状況を管理する。

---

# 2. システム構成

|項目|内容|
|---|---|
|Backend|Django|
|Frontend|HTML / CSS / JavaScript|
|DB|Oracle|
|参照View|OFCSEUC.FCSEUC_TRACE_SUMMARY_V|

---

# 3. 画面仕様

## 3.1 画面名

AOI長泉倉庫進捗ダッシュボード

---

## 3.2 画面レイアウト

### ヘッダー固定エリア

表示項目：

- 出荷日選択
- 集合梱包 進捗
- 釣銭機系 進捗
- 全体進捗

表示形式：

```
集合梱包
6 / 10 店舗
60%

釣銭機系
7 / 10 店舗
70%

全体
5 / 10 店舗
50%
```

### 詳細一覧エリア

一覧部分のみスクロール可能とする。

ヘッダー行は固定表示する。

---

## 3.3 一覧表示項目

|項目|内容|
|---|---|
|FFC出荷NO|出荷単位|
|店番|店舗番号|
|店舗名|店名|
|集合梱包 シリアル取得|完了数 / 予定数|
|釣銭機系 シリアル取得|完了数 / 予定数|
|全体ステータス|完了 / 未完|

---

## 3.4 色分け

### 集合梱包

完了数 = 予定数の場合

- 背景：緑色
- 文字：強調表示

---

### 釣銭機系

完了数 = 予定数の場合

- 背景：緑色
- 文字：強調表示

---

### 全体ステータス

|状態|表示|
|---|---|
|完了|緑色|
|未完|赤色|

---

## 3.5 ソート機能

以下の項目はクリックで並び替え可能とする。

対象：

- 集合梱包 シリアル取得
- 釣銭機系 シリアル取得
- 全体ステータス

並び順：

- 完了 → 未完
- 未完 → 完了

切替可能とする。

## 4.2 FFC案件条件

対象案件：

```text

```

```sql

# 6. 集計単位

## 6.1 画面表示単位

1行 = 1 FFC出荷NO

---

## 6.2 シリアル集計単位

枝番単位で集計する。

同一FFC出荷NO内で同じ枝番が複数存在する場合は
重複カウントしない。

例：

|FFC出荷NO|枝番|製品コード|
|---|---|
|F000200001|001|FFC00000005|
|F000200001|001|FFC00000006|
|F000200001|001|10021052698|

結果：

```
枝番001 = 1台
```

---

# 7. ステータス判定

# 9. 集合梱包集計

## 予定数

条件：

- 集合梱包対象製品
- 同一FFC出荷NO内の枝番を重複除外

集計：

```
COUNT(DISTINCT 枝番)
```

---

## 完了数

条件：

- 集合梱包対象製品
- ステータス（Fコード） = 作業終了
- ステータス（Fコード） = 削除 は集計対象外

同一枝番は重複除外してカウントする。

---

# 10. 釣銭機系集計

## 予定数

条件：

- 釣銭機系対象製品
- 同一FFC出荷NO内の枝番を重複除外

---

## 完了数

条件：

- 釣銭機系対象製品
- ステータス（Fコード） = 作業終了
- ステータス（Fコード） = 削除 は集計対象外

同一枝番は重複除外する。

---

# 11. 全体ステータス判定

以下の条件を満たす場合のみ完了。

```
集合梱包 完了数 = 集合梱包予定数

かつ

釣銭機系 完了数 = 釣銭機系予定数
```

判定：

|条件|結果|
|---|---|
|両方完了|完了|
|どちらか未完|未完|

---

# 12. 上部サマリー集計

集計単位：

FFC出荷NO（店舗単位）

---

## 集合梱包

```

計算：

```

÷
対象店舗数
```

---

## 釣銭機系

計算：

```
釣銭機系完了店舗数
÷
対象店舗数
```

---

## 全体

計算：

```
全体完了店舗数
÷
対象店舗数
```

# 14. 開発上の注意事項

- FFC案件NOは必ず F0002 のみ対象とする
- ステータス（FFC出荷NO）は使用しない
- ステータス判定はステータス（Fコード）のみ使用する
- シリアル数は枝番単位で集計する
- 同一枝番は重複カウントしない
- COUNT(*)で集計しない
- 画面は作成済みHTMLモックをベースにする
- PC表示を前提とする
- 詳細一覧のみスクロール可能とする
- DB接続不可の場合はデモデータで動作確認できる構成にする
